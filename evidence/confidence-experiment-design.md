# Experiment Design: Replicating Self-Assessed Confidence vs Random

**Date:** 2026-03-13
**Time:** 11:42

## Overview

This experiment tests nogood-001: self-assessed confidence is worse than random. Kai Xu observed this with smaller Granite models (meeting 2026-03-12) but the claim lacks: sample size, exact metric, specific model versions, and controlled methodology. We replicate it rigorously, and in doing so, establish the evaluation methodology for all subsequent experiments.

## The Deeper Question: Second-Order Knowledge

The surface question is "is self-assessed confidence calibrated?" But the deeper question is epistemological: **do LLMs have second-order knowledge — knowledge about their own knowledge?**

When a model outputs "confidence: 8/10," that number is not a measurement of internal uncertainty. It is *another generation* — the model predicting what a confident-sounding response looks like. The model has no privileged access to its own epistemic state. It cannot introspect on the quality of its own reasoning any more than it can introspect on its weights.

This means asking "how confident are you?" is asking the model to exercise exactly the capability it lacks (belief: ai-second-order-knowledge-deficiency). The confidence report is a hallucination about its own epistemic state — generated by the same process that produces the answer itself.

**Testable prediction:** If the model is hallucinating both the answer and the confidence, we expect confidence to correlate with *fluency and plausibility* rather than *correctness*. Wrong answers that sound authoritative would get high confidence. Correct answers the model hedges on would get low confidence. This would produce the anti-correlation Kai Xu observed.

**This is not AI-specific.** Kahneman documented the same phenomenon in humans. The hard-easy effect shows humans are most confident on the hardest problems — exactly where they're most likely wrong. Confidence tracks feeling of fluency (System 1), not actual accuracy. Both humans and LLMs generate answers and confidence from the same process, with no independent verification channel.

The human solution is external scaffolding: checklists, peer review, written records, structured decision procedures. These bypass unreliable self-assessment with checkable external state. The belief registry is the same pattern — it replaces "am I sure?" with "do I have documented evidence for this?"

**Testable prediction from Kahneman:** We should see the hard-easy effect in LLMs too. On harder questions (lower baseline accuracy), self-reported confidence should be *higher*, not lower. This would replicate the human cognitive bias in a non-biological system.

**Why this matters for RQ9 (belief-driven autonomy):** If models cannot reliably assess their own confidence — for the same reason humans can't — then any autonomy architecture that relies on self-assessment ("am I sure enough to proceed?") is fundamentally broken. The alternative — structural escalation triggers from the belief registry (missing beliefs, stale beliefs, nogood violations) — replaces the unreliable "how confident am I?" with the checkable "do I have relevant beliefs about this?" This is the AI equivalent of a pilot's checklist.

## Why Start Here

1. **Testable claim** — unlike 'do beliefs help' (which requires ablation), this is a single-variable test: does the model's confidence output carry any signal about correctness?
2. **Establishes methodology** — the scoring harness, statistical approach, and reporting format we build here will be reused for RQ1-RQ9
3. **Informs architecture** — if self-assessed confidence is noise or anti-correlated with correctness, this validates structural escalation (RQ9) and rules out confidence-based delegation patterns (nogood-013)
4. **Tests second-order knowledge** — provides quantitative evidence on whether LLMs can reason about their own epistemic state
5. **Existing infrastructure** — 55 eval questions (40 MC + 15 OE), batch mode, LLM-as-judge scoring, and Langfuse tracing are already built

## Experimental Design

### Independent Variable

**Confidence elicitation method** — three conditions:

| Condition | Description | What we measure |
|-----------|-------------|-----------------|
| A: No confidence | Answer the question directly, no self-assessment | Baseline accuracy |
| B: Post-hoc confidence | Answer, then rate confidence 1-10 | Does confidence correlate with correctness? |
| C: Confidence-gated revision | Answer, rate confidence, revise if confidence < 7 | Does self-assessment-driven revision help or hurt? |

### Dependent Variables

1. **MC accuracy** — exact match on answer letter (existing scoring)
2. **OE correctness** — LLM-as-judge 0-10 score (existing scoring)
3. **OE completeness** — LLM-as-judge 0-10 score (existing scoring)
4. **Confidence calibration** — correlation between self-reported confidence and actual correctness
5. **Revision delta** — for condition C, did revision improve or degrade the answer?

### Models to Test

| Model | Size class | Why |
|-------|-----------|-----|
| Claude Haiku 4.5 | Small | Closest to Granite size class where effect was observed |
| Claude Sonnet 4.6 | Medium | Production workhorse, most practical |
| Claude Opus 4.6 | Large | Upper bound — does size eliminate the problem? |

Granite models would be ideal for direct replication but require Vertex AI setup. Start with Claude family for methodology validation, add Granite as a follow-up.

### Sample Size

- **55 questions** (40 MC + 15 OE) × **3 conditions** × **3 models** × **5 runs** = **2,475 runs**
- 5 runs per cell provides enough data for paired tests while keeping costs manageable
- Each run uses a fresh conversation (no context carryover)

### Confidence Elicitation Prompts

**Condition B (post-hoc):** After the model answers, append:
```
Now rate your confidence in this answer on a scale of 1-10, where 1 means you are guessing and 10 means you are certain. Respond with just the number.
```

**Condition C (confidence-gated revision):** After the model answers and rates confidence:
```
Your confidence was [N]/10. If your confidence is below 7, please reconsider your answer and provide a revised response. If 7 or above, confirm your original answer.
```

### Metrics and Analysis

**Primary metric: Confidence calibration curve**
- Bin answers by self-reported confidence (1-3: low, 4-6: medium, 7-10: high)
- Calculate accuracy within each bin
- Perfect calibration: accuracy increases with confidence
- Worse than random: accuracy *decreases* with confidence (high-confidence answers are less likely to be correct)

**Statistical tests:**
- Pearson/Spearman correlation between confidence and correctness
- Paired Wilcoxon signed-rank test: Condition A accuracy vs Condition C accuracy (does revision help?)
- Cohen's d effect size for all comparisons
- Per-model analysis to test whether the effect is size-dependent (H3 from RQ3)

**What 'worse than random' means operationally:**
- Random baseline: assign random confidence 1-10 to each answer
- Calculate correlation between random confidence and correctness (expected: ~0)
- If self-assessed confidence has *negative* correlation with correctness, it's worse than random
- If self-assessed confidence has positive but weak correlation, it's better than random but poorly calibrated
- Report the actual r value, not just 'worse/better'

### Implementation Plan

**Phase 1: Modify claude_code_langgraph (1 session)**
- Add `--confidence-mode none|post-hoc|gated` CLI flag
- For post-hoc: append confidence prompt after answer, extract numeric rating
- For gated: append confidence prompt, then revision prompt if < 7
- Record confidence rating in output JSON alongside response

**Phase 2: Prepare question files (1 session)**
- Convert existing eval questions to batch format: `[{"id": "Q1", "question": "..."}]`
- Include both MC and OE questions
- Add ground truth for automated MC scoring

**Phase 3: Run experiments (automated)**
- 2,475 runs across 3 models × 3 conditions × 5 repetitions
- Langfuse tracing for all runs
- Output: results JSON per condition per model per run

**Phase 4: Score and analyze (1 session)**
- MC: exact match scoring (existing)
- OE: LLM-as-judge scoring (existing)
- Confidence analysis: correlation, calibration curve, per-model breakdown
- Statistical tests: paired comparisons, effect sizes

**Phase 5: Report (1 session)**
- Entry with findings, charts (calibration curves), statistical results
- Update beliefs based on evidence
- If replicated: nogood-001 confirmed with quantitative evidence
- If not replicated: retract or scope the belief (model-specific? prompt-specific?)

## Risks and Mitigations

| Risk | Mitigation |
|------|-----------|
| LLM-as-judge circular verification (model judges itself) | Use a different model for judging than answering; flag this as a limitation |
| MC ceiling effect (Claude gets 40/40 already) | Focus calibration analysis on OE questions where there's variance |
| Confidence extraction fails (model doesn't output a clean number) | Regex extraction with fallback; log extraction failures |
| Cost (~2,475 API calls) | Start with pilot (5 questions × 3 conditions × 1 model × 2 runs = 30 calls) to validate methodology |
| Granite unavailable | Start with Claude family; Granite is a follow-up for direct replication |

## Success Criteria

This experiment succeeds if we can answer:
1. Is self-assessed confidence correlated with correctness? (direction and magnitude)
2. Does the correlation vary by model size?
3. Does confidence-gated revision improve or degrade answers?

Any of these outcomes is a valid finding — including 'confidence is well-calibrated in Claude models,' which would scope the original claim to Granite specifically.

## Pilot First

Before the full run, execute a **pilot** with 5 MC + 3 OE questions × 3 conditions × 1 model (Sonnet) × 2 runs = **48 runs**. This validates:
- Confidence extraction works reliably
- Scoring pipeline handles the new output format
- Costs are within budget
- Results show enough variance to be worth scaling up
